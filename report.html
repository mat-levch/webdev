Planning

It has a header with a navigational bar. You can get access to every page of the website. If you need to go back to the main page, you can just push on the logo in the header or footer. It also has a responsive layout and a mobile menu if you look at the website on your phone which helps to navigate between pages

Implementation

1.I learned more about the SASS preprocessor and how to use it. I practiced naming in HTML. I learned almost all the information from the Ukrainian IT courses I am studying at. The examples are mostly generic, so I found a way to apply them in my project.
1.My work uses a SASS preprocessor, which makes stylization by CSS mush easier. Every section is separated and has its own CSS file. Afrerall i used a css miniminazer to compress all css files into one big css file.
I also use a icomoon.io to make a pack which consists of a code with all icons from my page.
I use GitHub to create a domain for my website and make it accessible to other people. In GitHub you can follow up on all your files, and you can see all changes and much more interesting and useful staff.
I have couple very usefull extensions from VSCODE, such as Live Sass Compiler, Auto close tag, Auto save and Live server
